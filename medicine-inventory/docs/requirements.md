# 薬在庫管理システム_要件定義・共通設計

## 1. セキュリティ要件
* **家族専用利用:** 外部からの不正アクセスを防ぎ、家族のみが操作できること。
* **利便性の両立:** 忙しい育児の合間に使うため、頻繁なログイン操作を不要にすること。

## 2. 認証・認可設計
* **認証方式:** 簡易パスワード（合言葉）＋ JWT（JSON Web Token）方式。
* **セッション管理:** * サーバー側で状態を持たないステートレスな認証。
    * ブラウザにJWTを保存し、以降のAPI通信のヘッダーに添付する。
* **有効期限（寿命）:** * 利便性を優先し、JWTの有効期限は「1ヶ月」とする。
    * 期限切れの場合のみ、再ログイン（合言葉の再入力）を求める。

## 3. 外部連携設計
* **Googleカレンダー連携:** * 「サービスアカウント」方式を採用。
    * バックエンドがシステム的に予定を登録・更新する。
    * 薬ごとの通知ラインを下回った際に、自動でアラート予定を書き込む。
